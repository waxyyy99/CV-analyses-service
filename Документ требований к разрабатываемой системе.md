## Введение

### Область применения системы
Система представляет собой веб-платформу для комплексной оценки и улучшения резюме, ориентированную на студентов, молодых специалистов и соискателей работы. Она использует искусственный интеллект (ИИ) для автоматического анализа резюме независимо от конкретной вакансии: оценка структуры, оформления, наполнения (навыки, опыт, образование), выявление пробелов и генерация персонализированных советов по улучшению. Предметная область — рынок труда, где соискатели часто сталкиваются с проблемами в самооценке и доработке резюме. Система повышает качество документов, увеличивая шансы на успешное трудоустройство. Оценка проводится по взвешенным критериям (структура — 10%, оформление — 15%, наполнение — 30%, навыки — 20%, языки — 10%, образование — 15%), с использованием NLP (NER, TF-IDF) и BERT для семантического анализа.

### Определения, сокращения, аббревиатуры
- **AI (Artificial Intelligence)**: Искусственный интеллект — технологии, имитирующие человеческий интеллект, включая машинное обучение.
- **API (Application Programming Interface)**: Интерфейс программирования приложений — набор протоколов для взаимодействия между системами.
- **OCR (Optical Character Recognition)**: Оптическое распознавание символов — технология извлечения текста из изображений или PDF.
- **NLP (Natural Language Processing)**: Обработка естественного языка — подраздел ИИ для анализа человеческого текста.
- **BERT**: Bidirectional Encoder Representations from Transformers — предобученная модель NLP для задач классификации и извлечения сущностей.
- **UC (Use Case)**: Сценарий использования — описание взаимодействия пользователя с системой.
- **UI (User Interface)**: Пользовательский интерфейс — визуальная часть системы для взаимодействия с пользователем.
- **JWT (JSON Web Token)**: Токен для аутентификации — стандартный механизм безопасной передачи информации.
- **Docker**: Инструмент контейнеризации для упаковки приложений и их зависимостей.
- **F1-Score**: Метрика качества модели — гармоническое среднее precision и recall.
- **NER (Named Entity Recognition)**: Распознавание именованных сущностей — извлечение ключевых элементов (навыки, даты опыта).
- **TF-IDF**: Term Frequency-Inverse Document Frequency — метод векторизации текста для анализа ключевых слов.
- **ATS**: Applicant Tracking System — системы отслеживания кандидатов для парсинга резюме.

## 1. Концепция (видение) решения

### Проблема или Возможность
Проблема: Соискатели часто создают резюме без объективной оценки качества (структура, оформление, полнота содержания), что приводит к отказам на этапе скрининга ATS. Возможность: Автоматизированная ИИ-оценка резюме с советами по улучшению позволит сократить время на самоанализ в 5–10 раз, повысив качество документов и удовлетворенность пользователей. Монетизация через премиум-функции (детальные отчеты).

### Vision (Видение)
После реализации продукта соискатели смогут самостоятельно превращать свое резюме в конкурентоспособный документ: загрузка — анализ — персонализированные советы по оформлению и наполнению, без зависимости от вакансий. Роль интеллектуального сервиса — ключевая: ИИ (fine-tuned BERT с NER) будет оценивать качество по критериям (структура, ключевые слова, пробелы) и генерировать actionable рекомендации, делая платформу "личным редактором резюме".

### Цели и метрики успеха

#### Бизнес-цели
- Увеличить количество активных пользователей на 50% в первый год за счет рекомендаций среди студентов.
- Монетизировать через подписку (расширенные советы) с целью достижения 10% конверсии.

#### Пользовательские цели
- Получить объективную оценку резюме и советы по улучшению оформления/наполнения.
- Выявить слабые места (например, отсутствие quantifiable достижений) без привязки к вакансии.

#### Метрики успеха
- **North Star Metric**: Количество успешно улучшенных резюме в неделю (цель: 1000+ по опросам пользователей).
- **Технические метрики**: Скорость ответа API (<500 мс), uptime (99.9%).
- **ML-метрики**: Accuracy, Precision, Recall, F1-Score для классификаторов (цель: F1-Score ≥ 0.85); MAE, RMSE для регрессии оценки качества (MAE < 0.10); CTR для рекомендаций (>20%).
- **Пользовательские метрики**: Время выполнения задачи (<2 минуты), коэффициент отказов (<5%), NPS (Net Promoter Score) > 70 из опросов удовлетворенности.

### Общее описание

#### Перспектива продукта
Система — самостоятельная веб-платформа (standalone), но с возможностью интеграции как модуль в карьерные сервисы (например, HH.ru или LinkedIn). Архитектура: микросервисная (backend API + ML-сервис), развернутая в облаке.

#### Характеристики пользователей
- **Роли**: Гость (анонимный анализ), Зарегистрированный пользователь (сохранение истории), Администратор (управление моделями).
- **Технический уровень**: Средний (студенты/специалисты, знакомые с веб-интерфейсами).
- **Языковые предпочтения**: Русский (основной для UI), английский (для анализа резюме).

#### Функциональность на высоком уровне
- Загрузка и парсинг резюме (PDF/текст).
- ИИ-оценка: анализ структуры, оформления, наполнения; выявление пробелов.
- Генерация советов (по оформлению, ключевым словам, quantifiable метрикам) и визуализация.
- Аутентификация и история анализов.

#### Ограничения
- **Законодательные**: Соответствие GDPR (анонимизация персональных данных в резюме).
- **Технические**: Ограничение на размер файла (≤5 МБ); поддержка только PDF/текста (без изображений).
- **Ресурсные**: Бюджет на GPU для дообучения ML-модели (учебный проект: локальный запуск).

#### Предположения и зависимости
- Доступны внешние API: Hugging Face для датасетов (AI-Powered Resume Screening Dataset, resume-5label-classification) и ML-моделей.
- Библиотеки: Node.js (Express), React (Material-UI), Python (Transformers, spaCy), Docker.
- Зависимости: Стабильный интернет для загрузки датасетов (~1000+ резюме с аннотациями качества).

## 2. Функциональные требования

### Описание пользовательских сценариев (Use Cases)
- **UC-диаграмма**: (В приложении: UML-диаграмма с акторами "Пользователь" и "Система", UC: "Загрузить резюме", "Оценить качество", "Получить советы").
- **Перечень UC**:
  1. UC-01: Регистрация/логин пользователя.
  2. UC-02: Загрузка резюме.
  3. UC-03: Автоматическая оценка качества (структура, наполнение).
  4. UC-04: Генерация советов по улучшению (оформление, содержание).
  5. UC-05: Просмотр истории анализов.

#### UC-01 - Регистрация пользователя

**Цель:** Позволить пользователю создать учетную запись для персонализированного использования системы.

**Актор:** Пользователь.

**Предусловия:** — Пользователь находится на странице регистрации. — Интернет-соединение активно.

**Основной поток:**

1. Пользователь открывает страницу регистрации.
2. Вводит имя, телефон, пароль и подтверждение пароля.
3. Нажимает кнопку «Зарегистрироваться».
4. Система проверяет корректность введённых данных.
5. При успешной проверке создается учетная запись.
6. Система перенаправляет пользователя на страницу авторизации.

**Альтернативный поток:** — Если телефон уже используется — система выводит сообщение «Пользователь с таким телефоном уже существует». — Если пароль не соответствует требованиям — сообщение «Пароль должен содержать не менее 6 символов».

**Постусловия:** — Создана новая учетная запись пользователя.

#### UC-02 - Загрузка резюме

**Цель:** Позволить пользователю загрузить резюме для последующего анализа и оценки качества.

**Актор:** Пользователь (гость или зарегистрированный).

**Предусловия:** — Пользователь авторизован или находится в гостевом режиме. — Интернет-соединение активно. — Файл резюме готов к загрузке (PDF или текст, ≤5 МБ).

**Основной поток:**

1. Пользователь открывает страницу загрузки резюме.
2. Выбирает файл резюме (PDF/текст) через drag-and-drop или кнопку "Выбрать файл".
3. Нажимает кнопку «Загрузить».
4. Система проверяет формат и размер файла (OCR для PDF).
5. При успешной проверке текст извлекается и сохраняется временно.
6. Система подтверждает загрузку и предлагает перейти к оценке.

**Альтернативный поток:** — Если файл превышает 5 МБ — система выводит сообщение «Файл слишком большой, максимум 5 МБ». — Если формат неверный — сообщение «Поддерживаются только PDF и текст; попробуйте другой файл». — При сбое OCR — сообщение «Невозможно извлечь текст, загрузите другой файл».

**Постусловия:** — Резюме загружено и готово к анализу; текст извлечён и анонимизирован.

#### UC-03 - Автоматическая оценка качества резюме

**Цель:** Автоматически оценить качество резюме по критериям (структура, оформление, наполнение, навыки, языки, образование) и выдать взвешенный балл.

**Актор:** Пользователь.

**Предусловия:** — Резюме загружено (из UC-02). — Пользователь авторизован (для сохранения истории). — ML-модель (BERT + NER) доступна.

**Основной поток:**

1. Пользователь нажимает кнопку «Оценить качество» на дашборде.
2. Система запускает NLP-анализ: извлечение текста, NER для навыков/опыта, TF-IDF для ключевых слов.
3. Модель оценивает по критериям (веса: структура 10%, оформление 15%, наполнение 30%, навыки 20%, языки 10%, образование 15%).
4. Вычисляется общий балл (0–10) и баллы по разделам.
5. Система отображает результаты в дашборде (графики, radar chart).
6. Пользователь может просмотреть детальный отчёт.

**Альтернативный поток:** — Если анализ занимает >2 сек — система показывает прогресс-бар. — При сбое модели — fallback на rule-based оценку (e.g., подсчёт ключевых слов); сообщение «Оценка на основе базовых правил». — Если текст не извлечён — возврат к UC-02 с ошибкой.

**Постусловия:** — Сформирован отчёт с баллами; сохранён в историю (если авторизован).

#### UC-04 - Генерация советов по улучшению

**Цель:** Сгенерировать персонализированные рекомендации по доработке резюме на основе выявленных пробелов в критериях оценки.

**Актор:** Пользователь.

**Предусловия:** — Оценка качества завершена (из UC-03). — Пользователь авторизован (для персонализации).

**Основной поток:**

1. Пользователь нажимает кнопку «Получить советы» в отчёте оценки.
2. Система анализирует пробелы (e.g., низкий балл по наполнению — <30%).
3. Генерирует 3–5 советов (e.g., "Добавьте quantifiable метрики в опыт: 'Увеличил продажи на 20%'").
4. Советы группируются по критериям (структура, навыки и т.д.).
5. Система отображает список советов с примерами и визуализацией (до/после).
6. Пользователь может скачать отчёт с советами (PDF).

**Альтернативный поток:** — Если нет пробелов (балл >8) — сообщение «Резюме в хорошем состоянии; вот общие tips». — При ошибке генерации — fallback на шаблонные советы; сообщение «Общие рекомендации доступны». — Если пользователь не авторизован — предложение зарегистрироваться для персонализации.

**Постусловия:** — Сгенерированы и отображены советы; обновлена история с рекомендациями.

#### UC-05 - Просмотр истории анализов

**Цель:** Позволить пользователю просмотреть предыдущие оценки и советы для отслеживания прогресса улучшения резюме.

**Актор:** Зарегистрированный пользователь.

**Предусловия:** — Пользователь авторизован. — В истории есть хотя бы один анализ (из UC-03/04). — Интернет-соединение активно.

**Основной поток:**

1. Пользователь переходит в раздел «История» в меню.
2. Система загружает список анализов (дата, общий балл, краткое описание).
3. Пользователь выбирает запись для просмотра.
4. Система отображает полный отчёт: баллы по критериям, советы, сравнение с предыдущими (e.g., прогресс +15%).
5. Пользователь может скачать или удалить запись.
6. Система обновляет список после действий.

**Альтернативный поток:** — Если история пуста — сообщение «Нет сохранённых анализов; загрузите первое резюме». — При ошибке загрузки — retry; сообщение «Проверьте соединение и попробуйте снова». — При удалении — подтверждение: «Вы уверены? Удаление необратимо».

**Постусловия:** — История просмотрена/обновлена; данные сохранены в БД.

### Требования к функциональным модулям
Детальное описание функций системы, разделенное по модулям или компонентам. Для интеллектуальной веб-системы может включать:
- **Аутентификация и авторизация**: Регистрация, вход, управление ролями. Модуль: Node.js с bcrypt.
- **Обработка данных**: Сбор, очистка, анализ данных. Модуль: Python с pdfplumber и spaCy.
- **Интеллектуальные функции**: Алгоритмы машинного обучения, рекомендации, предсказания. Модуль: FastAPI с Hugging Face (fine-tuned BERT).
- **Визуализация**: Отчеты, дашборды, графики. Модуль: React + Chart.js.
- **Интеграции**: API, внешние сервисы. Модуль: Axios в frontend.

Каждый модуль описывается отдельно:
- **Аутентификация и авторизация**: Регистрация (email/пароль), JWT-токены, роли (user/admin).
- **Обработка данных**: Сбор (загрузка PDF), очистка (OCR + предобработка текста), анализ (NLP для извлечения компонентов).
- **Интеллектуальные функции**: Оценка качества (NER + scoring по критериям: длина, ключевые слова, структура), рекомендации (генерация советов на основе пробелов).
- **Визуализация**: Дашборд с графиками (баллы по разделам, radar chart для сильных/слабых сторон).
- **Интеграции**: API для датасетов (Hugging Face).

### Обработка ошибок и исключений (опционально)
Как система реагирует на неудачи (плохое качество данных, сбои подключения к ИИ-моделям и т.д.):
- При плохом PDF (OCR-сбой): Вернуть сообщение "Невозможно извлечь текст, загрузите другой файл" (HTTP 400).
- Сетевой сбой: Retry (3 попытки) для API-вызовов.

## 4. AI/ML Специфика

### Цели использования ИИ
Что именно делает ИИ: классификация (оценка качества по критериям), генерация (рекомендации по улучшению), прогнозирование (вероятность успеха резюме), извлечение информации (NER для навыков, опыта). Оценка по взвешенным критериям: структура (10%), оформление (15%), наполнение (30%), навыки (20%), языки (10%), образование (15%).

### Требования к данным
- **Источники данных**: Kaggle; Hugging Face.
- **Объём и формат данных**: 1000–5000 резюме с аннотациями качества; формат CSV/JSON.
- **Характеристики качества данных**: Чистый текст (без шума), аннотации по критериям (структура, ключевые слова), анонимизированные персональные данные.

### Метрики качества моделей
- Accuracy, F1-score, Precision/Recall.
- Минимальный порог точности: 85% (Accuracy, F1-score, Precision/Recall ≥80%).

### Жизненный цикл модели
- **Периодичность переобучения**: Ежемесячно на новых данных (обновления датасетов).
- **Валидация**: Cross-validation (5-fold); мониторинг drift.
- **A/B-тестирование**: Сравнение версий модели (zero-shot baseline vs. fine-tuned) по F1-score.

### Этические и правовые аспекты
- **Объяснимость (Explainability)**: SHAP для интерпретации оценок (почему раздел "навыки" слабый).
- **Bias / Fairness**: Проверка на гендер/возрастный bias (fairlearn); баланс датасета.
- **GDPR и пр.**: Анонимизация данных, согласие на обработку, хранение <30 дней.

1. **Диаграммы (UML, BPMN, архитектура)**: UML UC-диаграмма: Актор "Пользователь" → UC "Загрузить резюме" → "Оценка ИИ" → "Советы". Архитектура: Микросервисы (Node.js API → Python ML → PostgreSQL).
2. **Прототипы интерфейсов**: Figma-скетчи: Экран загрузки, экран оценки (radar chart для критериев, список советов).
